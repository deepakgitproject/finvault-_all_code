================================================================================
                          RUNNING SPECIFIC TESTS
================================================================================

# Run tests by class name
dotnet test --filter "FullyQualifiedName~UserTests"

# Run tests by method name
dotnet test --filter "FullyQualifiedName~Create_WithValidParameters"

# Run tests by category
dotnet test --filter "Category=Domain"

# Run tests matching a pattern
dotnet test --filter "DisplayName~ShouldCreate"

# Run a single test
dotnet test --filter "Name=Create_WithValidParameters_ShouldCreateUser"

================================================================================
                          TEST OUTPUT OPTIONS
================================================================================

# Normal output (default)
dotnet test

# Verbose output with all test names
dotnet test --logger "console;verbosity=detailed"

# Minimal output (only failures)
dotnet test --logger "console;verbosity=minimal"

# Output to file
dotnet test --logger "trx;LogFileName=test-results.trx"

# HTML report (requires ReportGenerator)
dotnet test --collect:"XPlat Code Coverage"
reportgenerator -reports:"**/coverage.cobertura.xml" -targetdir:"CoverageReports"

================================================================================
                          CODE COVERAGE
================================================================================

# View coverage in Visual Studio

# Generate coverage report
dotnet test --collect:"XPlat Code Coverage" -- DataCollectionRunSettings.DataCollectors.DataCollector.Configuration.Format=cobertura

# Open coverage report
start CoverageReport\index.html

================================================================================
                          TROUBLESHOOTING
================================================================================

Problem: Tests not found
Solution: 
  dotnet restore
  dotnet build
  dotnet test

Problem: Tests failing due to database
Solution:
  # Tests use mocks, no database needed for unit tests
  # Check if mock setup is correct

Problem: Code coverage not generating
Solution:
  dotnet tool install --global coverlet.console
  coverlet "bin/Debug/net9.0/Tests.dll" --target "dotnet" --targetargs "test --no-build"

Problem: Tests timing out
Solution:
  # Increase timeout in test
  [Test, Timeout(10000)]  // 10 seconds
  public async Task LongRunningTest() { ... }

================================================================================
                          BEST PRACTICES
================================================================================

1. Run tests before every commit
2. Keep tests fast (< 1 second per test for unit tests)
3. Tests should be independent and isolated
4. Use descriptive test names (Given_When_Then format)
5. One assertion per test when possible
6. Mock all external dependencies
7. Don't test third-party libraries
8. Review tests in pull requests
9. Maintain test coverage above 75%
10. Fix failing tests immediately
